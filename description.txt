connect.php
Назначение: подключение к базе данных PostgreSQL.

Что делает:

Устанавливает соединение с базой с помощью pg_connect().

Если подключение не удалось — выводит сообщение и завершает скрипт.

Ключевой элемент:

$conn = pg_connect("host=localhost dbname=univer user=postgres password=..."); 
Где используется: подключается (include) в каждом скрипте, который взаимодействует с базой.

pers.php
Назначение: выводит таблицу pers (персональные данные студентов).

Что делает:
Выполняет SELECT * FROM pers
Формирует HTML-таблицу со всеми полями: ID, дата рождения, город, адрес, зачетка, логин, пароль.
Используется для: простого просмотра всех студентов.

fo.php
Назначение: выводит ФИО студентов из таблицы fo.
Что делает:
Выполняет SELECT * FROM fo
Показывает таблицу с полями: ID, фамилия, имя, отчество.

professors.php
Назначение: выводит преподавателей.
Что делает:
Выполняет SELECT * FROM professors
Показывает таблицу с фамилией, именем, отчеством.

subjects.php
Назначение: выводит список предметов.
Что делает:
Выполняет SELECT * FROM subjects
Показывает название предмета и ID преподавателя.

examen.php
Назначение: показывает список экзаменов.
Что делает:
Выполняет SELECT * FROM examen
Показывает ID студента, ID предмета, дату и оценку.

search_form.html
Назначение: HTML-форма для поиска информации по студенту.
Что делает:
Пользователь вводит ФИО студента
Выбирает, что получить: персональные данные / экзамены / предметы
Отправляет GET-запрос на form_handler.php
Важно: три radio-кнопки управляют логикой, какой файл будет вызван (sql1, sql2, sql3).

form_handler.php
Назначение: обработка формы search_form.html.
Что делает:
Получает surname, name и action из формы
В зависимости от значения action, подключает нужный скрипт:
if ($action == "1") include "sql1.php";
elseif ($action == "2") include "sql2.php";
elseif ($action == "3") include "sql3.php";

sql1.php
Назначение: выводит персональные данные по ФИО студента.
Что делает:
Получает ID студента из fo по ФИО
Использует этот ID, чтобы выбрать строки из pers
Формирует таблицу с персональными данными
Вызов include "sql.php" рендерит сами строки

sql2.php
Назначение: выводит экзамены, которые сдавал студент.
Что делает:
По ФИО → ищет ID студента в fo
Через examen JOIN subjects показывает: ID, предмет, дата, оценка
Также использует sql.php для рендеринга строк

sql3.php
Назначение: показывает предметы и преподавателей, по которым у студента были экзамены.
Что делает:
Через examen → subjects → professors вытаскивает:
название предмета
фамилию и имя преподавателя
Также использует sql.php для универсального вывода строк

sql.php
Назначение: универсальный шаблон вывода строк таблицы.
Что делает:
Получает результат из переменной $sql
Выполняет pg_query(), перебирает строки и ячейки
Автоматически рендерит любую таблицу (без знания структуры)
Вызывается в: sql1.php, sql2.php, sql3.php

insert_form.html
Назначение: форма для добавления нового студента (в fo и pers).
Что делает:
Позволяет ввести ФИО, дату рождения, адрес, город, номер зачетки, логин, пароль
Отправляет данные в sqlinsert.php по GET-запросу

sqlinsert.php
Назначение: вставляет нового студента в fo и pers.
Что делает:
Автоматически рассчитывает id = max(id) + 1
Вставляет строку в fo
Вставляет строку в pers с тем же ID
Особенности:
Учитывает pg_query($conn, ...)
Проверяет ограничения PostgreSQL по типам (например, INT)

insert_subject.html
Назначение: форма для добавления нового предмета в subjects.
Что делает:
Вводится название предмета и ID преподавателя
Отправляется GET-запрос в insert_subject.php

insert_subject.php
Назначение: обработчик для добавления строки в subjects.
Что делает:
id_subject = max(id_subject) + 1
Вставка в subjects(subject_name, id_prof)

insert_exam.html
Назначение: форма для добавления экзамена (в examen).
Что делает:
Вводится ID студента, ID предмета, дата и оценка
Данные передаются в insert_exam.php

insert_exam.php
Назначение: вставка экзамена в таблицу examen.
Что делает:
ID экзамена = max(id) + 1
Вставка: id, id_subject, date, result
Особенность: здесь id экзамена совпадает с ID студента (по структуре таблицы).
